Overview

Pymatrix is a lightweight terminal application that recreates the iconic "digital rain" effect from The Matrix. Written in pure Python using only the standard library, it runs directly in your Linux terminal with no external packages required.

Like cmatrix and tmatrix, pymatrix renders cascading glyphs in columns across your terminal — but it's written entirely in Python, making it trivially readable, forkable, and hackable.

// NOTE

Pymatrix uses Python's curses module for terminal control. This module is part of the Python standard library and ships pre-installed on all major Linux distros and macOS.

Requirements

Pymatrix has intentionally minimal requirements. If you have Python installed, you almost certainly have everything already.

PackageVersionNotes
Python3.6+Core runtime. Any modern distro has this.
python3-cursesstdlibTerminal manipulation. Usually pre-installed.
Ubuntu / DebianAny LTSTested on Ubuntu 22.04 and 24.04.
Verify your setup
python3 --version python3 -c "import curses; print('curses OK')"

If the curses check fails, install it manually:

sudo apt-get install python3 python3-curses
Installation
Method 1 — Clone via Git (recommended)
git clone https://github.com/nihalsheikh/pymatrix.git cd pymatrix
Method 2 — Download ZIP

Go to github.com/nihalsheikh/pymatrix, click Code → Download ZIP, then extract and cd into the folder.

Method 3 — wget
wget https://raw.githubusercontent.com/nihalsheikh/pymatrix/main/pymatrix.py python3 pymatrix.py
// TIP

There's nothing to install — no pip install, no setup.py, no package index. Clone and run. That's the entire install process.

Running

Two equally valid ways to run pymatrix from inside the cloned directory:

Method A — Python directly
cd pymatrix python3 pymatrix.py
Method B — Make executable
cd pymatrix chmod +x pymatrix.py ./pymatrix.py
// WINDOWS WARNING

Pymatrix uses the curses module which is not available on Windows by default. This project targets Linux terminals. WSL (Windows Subsystem for Linux) should work but is not officially tested.

Recommended terminal setup

For the best visual experience, use a terminal that supports:

SettingRecommended ValueWhy
FontMonospace (Fira Code, JetBrains Mono, Gohu)Equal-width characters for aligned columns
Background#000000 pure blackMaximum contrast for green rain
Terminal sizeFullscreen / maximizedMore columns = better effect
Color support256-color or 24-bitSmooth green gradient in trails

See Terminal Configuration for per-terminal setup instructions.

Controls

Pymatrix has two exit controls. There is no other interaction — the animation runs and you watch.

KeyAction
qQuit gracefully. Cleans up the terminal state properly — your shell will look normal after.
Ctrl+CForce exit. Works but may leave your terminal in a weird state. Use reset if needed.
// IF YOUR TERMINAL BREAKS

If you force-quit and your terminal looks garbled, type reset and press Enter. This restores your terminal to its default state.

Terminal Configuration

Pymatrix looks best with the right terminal setup. Here are configs for popular Linux terminals.

Kitty
background #000000 foreground #00ff41 font_family JetBrains Mono font_size 13.0
GNOME Terminal / Ubuntu Terminal

Preferences → Profiles → Your profile → Colors → uncheck "Use colors from system theme" → set Background to #000000.

Alacritty
colors: primary: background: '0x000000' foreground: '0x00ff41'
Gohu Font (recommended)

The screenshots in the repo use Gohu font — a bitmap monospace font designed for terminal use. Install on Ubuntu:

sudo apt-get install fonts-gohu

For full terminal configuration details, see Terminal_Configuration.md in the repo.

Troubleshooting
ProblemFix
ModuleNotFoundError: cursesRun: sudo apt-get install python3-curses
Terminal looks garbled after exitType reset and press Enter
Colors look wrong / no greenCheck your terminal supports 256-color: echo $TERM should show xterm-256color
Characters misaligned / brokenUse a monospace font. Variable-width fonts break column alignment
Runs too fast / too slowSpeed control is on the roadmap — coming in a future version
Script not found errorMake sure you're inside the pymatrix directory: cd pymatrix
